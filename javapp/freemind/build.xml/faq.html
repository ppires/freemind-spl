Installation

How can I install Freemind?

 FreeMind requires Java 1.4 or higher If Java is not installed, please abort the installation and download the JRE - Java Runtime Environment from here.

I installed Java JRE and FreeMind, execute freemind.exe (or the icon with the butterfly) and nothing happens. Why?

 Your Java installation is buggy. Try to excecute using "Run command..." in the start menu the command "javaw.exe". If this fails, you have to arrange your path settings, such that the java/bin directory is in your path.

Or try to start the batch "freemind.bat", read the messages and react.

Usage

How can I create a graphical link?
  1. Go to the source node.
  2. Mark it with the mouse.
  3. Press the left mouse button.
  4. Hold this button.
  5. Press Ctrl+Shift buttons.
  6. Continue to hold all these buttons.
  7. Go to the destination node.
  8. Release the mouse button, first.
  9. Then release the keybord buttons.
  10. You have your link.

How can I embed images into freemind?

 This is contained in a context menu using right click (or whatever this is on a MAC) on a node. Or type the following:

<html><img src="image.jpg"></html>

where image.jpg is the reference to an image.

How can I present my map in the web?

Use the freemind browser applet. The following document (thanks to Bob) explains what to do to have a complete testing environment for your applet. The most common problem to see a freemind browser applet is that you have to use Java >= 1.4 in your browser. Check this using the "About plugins" section of your browser (or a similar place).

Customization

Using another browser

 If you want to use another browser, edit the user.properties file and change the following property according to your operation system:
  • default_browser_command_windows_nt
  • default_browser_command_windows_9x
  • default_browser_command_mac
  • default_browser_command_other_os
You have to write something like
  • default_browser_command_windows_nt = C:\Program Files\Internet Explorer\iexplore.exe "{0}"

Editing the user properties

 The user.properties file resides in a subdirectory called "freemind" of your home directory. As an example, under Windows 2000, the file is normally in:
  • C:\Documents and Settings\<username>\freemind

Changing the language

 You have to edit your user.properties and to replace the line saying:
  • #language = en
With the line:
  • language = de
(You have to remove the '#', too. That is important.) Now, restart FreeMind and you have changed the language to german. Other available languages are french (fr), danish (dk), spanish (es), italian (it), japanese (ja).
If your language isn't supported yet, please translate the file Resource_en.properties and send the result to the maintainers.

FreeMind on the Mac

 OK, FreeMind runs great as there is an own download version for it. But you have the problem, that there is no "insert" key on the Mac. The solution is the following:
You have to edit your user.properties and to replace the line saying:
  • #keystroke_add_child = INSERT
With the line:
  • keystroke_add_child = control shift ENTER
(You have to remove the '#', too. That is important.) Now, you can add a new child to a node using the "control shift ENTER" key (or whatever you like and put instead of the above).

Common Problems

If there are more than one link in the same area, I cannot change the properties of the link

 The reason is that right clicking these links, the first link is taken that intersects a rectangle around the cursor. To get the other link(s), you have to find a place without other interfering links or you have to fold other links, such that they do not disturb.

What do I need to do to become a developer?

 Due to project reasons, we cannot make you a developer with CVS access of FreeMind directly. You have to follow the following steps:
  • Generate a Sourceforge account.
  • Then, start to discuss the change/feature/bug in our open discussion forum.
  • If the topic is discussed and hopefully accepted, start to change the actual CVS code (see on this page, how to compile Freemind from CVS).
  • At an early stage of your implementation (and testing), put a version into the patch section of sourceforge and/or send it directly to the current project manager. As this is an open project it may be the case that you have to wait a little time for the first responses, but you should get support, comments and help from the project members soon.
  • After having successfully developed and integrated some items, you get the developer rights at the sourceforge page to continue our good work. But this should not change the steps described above.
